## Banking77 StackEval / GEPA

### `banking77-policy-api-key-env`

StackEval Banking77 can fail before optimization if the policy provider env is
missing or points at an unsupported Chat Completions route. Harness presets
should preflight policy env separately from local optimizer health.

### `gepa-heldout-budget`

A smoke GEPA run proves plumbing, not prompt improvement. Gate presets need
enough train/heldout samples, generations, proposals, and seed-vs-best heldout
measurement to support an uplift claim.

### `gepa-candidate-registry-list-shape`

`candidate_registry.json` may be list-shaped. Harvest code must parse the
registry shape actually emitted by local GEPA instead of assuming a dict.

### `stackeval-stale-grade-output`

StackEval grader/reviewer stages must remove stale output files before invoking
Codex, otherwise old `grade.json` or `review.json` can create a false pass.
